# 上下文加载策略

这个 skill 会被反复用于很多不同的新对话里。

最大的风险不是“不会做”，而是每次都把整个知识库重新读一遍，浪费上下文并拖慢处理速度。

所以默认原则是：

- 只读取当前任务真正需要的规则和内容
- 不重复读取同一对话中已经确认过的上下文

## 一、Bootstrap 分级

### Level 0：不额外读取

适用场景：

- 当前对话里已经加载过相关规则
- 当前对话里已经定位过相关页面
- 用户只是继续上一步操作

例子：

- “就放到刚才那个 OpenClaw 主题下面”
- “把刚才那篇删掉”

### Level 1：只读规则

适用场景：

- 这是一个新对话
- 任务涉及分类、索引、管理规则
- 不确定这次该不该把某个对象放进 `Knowledge Index`

推荐读取：

- `Knowledge Index 使用说明`
- 如果涉及结构，再读取 `Knowledge Hub`

### Level 2：只读目标内容

适用场景：

- 任务围绕一个主题或一篇文章展开
- 属于新增、更新、查询、局部清理

执行方式：

- 搜索目标标题、主题或分支
- 只读取命中的页面和相关索引行

禁止做法：

- 不要顺手扫描整个知识库
- 不要为了一个局部任务加载无关分支

### Level 3：读取结构上下文

适用场景：

- 调整结构
- 重整分类
- 删除文章并清理结构
- 修复一个主题分支的关联关系

执行方式：

- 读取目标 topic hub
- 读取相关索引行
- 读取可能被影响的关联页面

## 二、什么时候不要重复读取规则

同一对话中，如果已经读取过以下内容，就不要重复读取：

- `Knowledge Index 使用说明`
- `Knowledge Hub`
- 当前正在处理的分支页面

除非：

- 用户明确说规则变了
- 用户明确说结构变了
- 当前任务依赖一个之前从未读取过的规则

## 三、一个简单判断方法

执行前先问自己三件事：

1. 我这次是在改“系统规则”，还是只是“用系统做事”？
2. 我这次处理的是一篇文章、一个主题，还是整个分支？
3. 当前对话里已有的信息，是否足够支撑这次任务？

如果第 3 个问题答案是“足够”，就不要再多读。
