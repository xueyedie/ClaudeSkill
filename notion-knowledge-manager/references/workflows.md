# 工作流说明

## 一、新增：从 URL 或 Markdown 入库

步骤：

1. 读取来源内容
2. 判断所属 `Domain`
3. 判断所属 `Parent Topic`
4. 搜索已有 canonical 页面和近邻页面
5. 决定：
   - 补充到现有页面
   - 新建子页面
   - 新建独立页面
6. 更新内容页
7. 更新或创建索引行
8. 向用户说明放置结果和原因

## 二、重整：整理某个分支

步骤：

1. 只读取目标分支，不默认读全库
2. 检查命名、归类、重复、canonical 缺失和关联失真
3. 先给出最小变更方案
4. 执行用户确认的更新
5. 修正索引与关联关系

## 三、查询：找相关文章

步骤：

1. 先查索引
2. 排序优先级：
   - 同 `Domain`
   - 同 `Parent Topic`
   - canonical 优先
   - `Confidence` 高的优先
3. 当索引信息不足时再读正文
4. 返回链接并给出一句话相关性说明

## 四、删除：删除并清理

步骤：

1. 确定目标页面与对应索引行
2. 判断它属于内容层还是管理层
3. 删除或移动内容页
4. 删除或更新索引行
5. 清理 `Related Pages`、来源引用和附近结构
6. 输出简短审计说明

## 五、建议的后续自动化

这些不是 skill 的硬依赖，但很适合后续补上。

### 每周索引体检

检查：

- `Domain` 缺失
- `Parent Topic` 缺失
- `URL` 缺失
- `Last Reviewed` 过旧

### 每月结构巡检

检查：

- 重复页面
- 命名不一致
- 过大的 canonical 页面
- 应拆分或合并的分支

### Inbox 分流

把 `Inbox` 里的内容转成：

- 推荐落位
- 页面草稿
- 索引草稿

## 六、索引体检

默认应优先使用 Notion MCP 直接读取 `Knowledge Index` 做巡检，而不是先导出 CSV。

推荐顺序：

1. 读取规则页
2. 读取 `Knowledge Hub`
3. 读取 `Knowledge Index` 数据库与视图
4. 锁定异常分支或异常条目
5. 输出每周巡检报告

详细说明见 [inspection.md](inspection.md)。

## 七、CSV 补充方案

如果当前环境不便直接做全量巡检，或用户需要离线扫描，可使用：

- `scripts/index_hygiene.py`

详细说明见 [scripts.md](scripts.md)。
