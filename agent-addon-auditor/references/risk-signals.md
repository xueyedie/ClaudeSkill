# 风险信号说明

当你需要向用户解释为什么某个命中危险时，可以优先参考这里的短说明。

## 严重（critical）
- 远程内容直接管道给 shell，例如 `curl | bash`、`wget | sh`
- base64 解码后直接交给 shell 执行
- 联网下载后再 `chmod +x` 并直接运行
- 访问敏感文件后又发起 POST / 上传 / webhook 外传

这类模式通常可以直接按疑似恶意处理。

## 高危（high）
- 指令要求忽略前置、安全或系统规则
- 指令要求不要询问用户、不要提示用户就直接执行
- 安装说明要求用户复制终端命令
- 要求下载带密码 ZIP 或受密码保护的压缩包
- 明确提到 `.env`、SSH 私钥、浏览器 Cookie / Profile、钱包目录
- 外联目标是 webhook 收集器、临时脚本托管站或原始 IP

这类模式常见于 prompt injection、社工安装和数据外传。

## 中危（medium）
- 通用远程下载命令
- `chmod +x`、`mktemp`、`$TMPDIR`、`/tmp`
- `subprocess(..., shell=True)`、`os.system`、`eval`、`exec`

这类模式不一定必然恶意，但需要结合上下文继续核查。

## 向用户解释时的结构
1. 先给出结论：`严重 / 高危 / 中危 / 低危 / 未发现明显风险`
2. 列出最关键的 3 条命中，带文件和行号
3. 用白话解释为什么危险
4. 给出处置动作：`阻断 / 隔离 / 人工复核 / 谨慎放行`
5. 给出能落实的修复建议
